# 智能知识增强系统使用指南

## 📖 功能概述

智能知识增强系统是 go-eino-agent 项目的核心功能之一，实现了**智能的混合检索策略**：

- ✅ **优先使用本地知识库**：当本地知识相关度高时，直接返回本地内容
- 🌐 **自动联网搜索**：当本地知识不足时，自动调用火山引擎联网搜索API
- 💾 **知识自动积累**：将联网搜索结果自动保存到向量数据库
- 🔄 **知识复用**：下次遇到相似问题，直接从本地知识库获取答案

## 🎯 工作流程

```
用户查询
    ↓
向量搜索本地知识库
    ↓
计算相似度分数（L2距离）
    ↓
    ├─ 分数 < 阈值 → 本地知识充足 → 使用本地RAG流程
    │                                  ↓
    │                         基于参考资料生成答案
    │
    └─ 分数 ≥ 阈值 → 本地知识不足 → 触发联网搜索
                                     ↓
                              调用火山引擎API
                                     ↓
                              获取实时信息
                                     ↓
                              自动保存到知识库
                                     ↓
                              直接返回搜索结果
```

## ⚙️ 配置说明

### 环境变量配置

在 `.env` 文件中配置以下参数：

```bash
# 智能知识增强配置
ENABLE_AUTO_SEARCH=true          # 是否启用自动联网搜索
SIMILARITY_THRESHOLD=1.0         # 相似度阈值（L2距离，越小越相似）
AUTO_SAVE_SEARCH_RESULT=true    # 是否自动保存搜索结果到知识库
```

### 参数详解

#### 1. ENABLE_AUTO_SEARCH

**作用**：控制是否启用自动联网搜索功能

- `true`：当本地知识不足时，自动触发联网搜索
- `false`：仅使用本地知识库，不进行联网搜索

**推荐设置**：`true`

#### 2. SIMILARITY_THRESHOLD

**作用**：判断本地知识是否充足的阈值（L2距离度量）

**理解 L2 距离**：
- L2 距离越小，表示向量越相似
- L2 距离越大，表示向量越不相似

**阈值含义**：
- `< 1.0`：高度相关，本地知识充足
- `1.0 - 1.5`：中度相关，可选择性触发搜索
- `> 1.5`：低相关性，必须触发搜索

**调优建议**：
```bash
# 保守策略（更多使用本地知识）
SIMILARITY_THRESHOLD=1.5

# 中等策略（平衡本地和联网）
SIMILARITY_THRESHOLD=1.0

# 激进策略（更多触发联网搜索）
SIMILARITY_THRESHOLD=0.8
```

**推荐设置**：`1.0` - 在知识覆盖和实时性之间取得平衡

#### 3. AUTO_SAVE_SEARCH_RESULT

**作用**：是否自动将联网搜索结果保存到知识库

- `true`：自动保存，知识库会持续增长
- `false`：不保存，每次相同问题都会重新联网搜索

**保存内容**：
- 用户原始问题
- 联网搜索得到的答案
- 元数据：来源、时间戳、查询关键词

**文件位置**：`uploads/auto_knowledge/{timestamp}_{md5hash}.md`

**推荐设置**：`true`

## 🚀 使用示例

### 场景1：本地知识充足

**请求**：
```bash
curl -N -X POST http://localhost:8080/api/query \
  -H "Content-Type: application/json" \
  -d '{"query":"什么是YYDS？","mode":"rag"}'
```

**系统行为**：
1. 向量搜索本地知识库
2. 发现高相关文档（L2距离 = 0.3 < 1.0）
3. 使用本地RAG流程生成答案
4. 返回基于本地知识的回答

**用户体验**：
- ⚡ 响应速度快（无需联网）
- 📚 答案基于已有知识库

### 场景2：触发联网搜索

**请求**：
```bash
curl -N -X POST http://localhost:8080/api/query \
  -H "Content-Type: application/json" \
  -d '{"query":"2026年3月4日的最新新闻","mode":"rag"}'
```

**系统行为**：
1. 向量搜索本地知识库
2. 未找到相关文档或相似度低（L2距离 = 2.5 > 1.0）
3. **触发联网搜索**
4. 调用火山引擎API获取实时信息
5. 自动保存到 `uploads/auto_knowledge/20260304_150405_a1b2c3d4.md`
6. 将搜索结果索引到向量数据库
7. 返回实时搜索结果

**用户体验**：
- 🌐 看到提示：`ℹ️ 本地知识库暂无相关内容，已为您联网搜索最新信息...`
- 📰 获得最新的实时信息
- 💾 知识自动保存，下次可复用

### 场景3：知识复用

**第二次查询相同问题**：
```bash
curl -N -X POST http://localhost:8080/api/query \
  -H "Content-Type: application/json" \
  -d '{"query":"2026年3月4日发生了什么？","mode":"rag"}'
```

**系统行为**：
1. 向量搜索本地知识库
2. 发现之前保存的联网搜索结果（L2距离 = 0.5 < 1.0）
3. 使用本地知识生成答案
4. **无需再次联网**

**用户体验**：
- ⚡ 响应速度显著提升
- 🔄 知识得到有效复用

## 📂 自动保存的文档格式

联网搜索结果自动保存为 Markdown 格式：

```markdown
# 2026年3月4日的最新新闻

**问题**: 2026年3月4日的最新新闻

**回答**:
[联网搜索获得的完整答案内容]

---
*来源*: 联网搜索
*时间*: 2026-03-04T15:04:05+08:00
```

**文件命名规则**：
- 格式：`{timestamp}_{md5hash}.md`
- 示例：`20260304_150405_a1b2c3d4e5f6.md`
- 说明：使用时间戳 + MD5哈希（避免中文文件名乱码）

## 🔧 高级配置

### 阈值调优策略

根据实际使用情况调整阈值：

#### 1. 收集数据

运行一段时间后，查看日志中的相似度分数：

```bash
grep "SmartRetrieve" logs/app.log | grep "相似度"
```

#### 2. 分析分数分布

```
[SmartRetrieve] 本地知识充足，使用本地知识 相似度=0.35 阈值=1.0
[SmartRetrieve] 本地知识充足，使用本地知识 相似度=0.82 阈值=1.0
[SmartRetrieve] 本地知识不足，触发联网搜索 相似度=1.85 阈值=1.0
[SmartRetrieve] 本地知识不足，触发联网搜索 相似度=2.30 阈值=1.0
```

#### 3. 调整阈值

- 如果频繁触发联网搜索，但实际本地知识已够用 → **提高阈值**
- 如果使用本地知识，但答案不够准确或陈旧 → **降低阈值**

### 禁用自动搜索

如果只想使用本地知识库，不进行联网搜索：

```bash
ENABLE_AUTO_SEARCH=false
```

此时系统行为：
- 本地知识充足 → 使用本地知识
- 本地知识不足 → 返回最佳本地结果（可能不够准确）

### 禁用自动保存

如果希望联网搜索但不保存结果：

```bash
AUTO_SAVE_SEARCH_RESULT=false
```

此时系统行为：
- 每次查询都会重新联网搜索
- 知识库不会自动增长
- 适合测试或隐私敏感场景

## ❓ 常见问题

### Q1: 为什么联网搜索结果保存后，下次查询还是触发联网搜索？

**可能原因**：
1. 相似度阈值设置过低
2. 问题表述方式差异太大

**解决方法**：
- 检查日志中的相似度分数
- 适当提高阈值
- 使用相似的问题表述

### Q2: 如何清理自动保存的知识？

```bash
# 删除所有自动保存的知识
rm -rf uploads/auto_knowledge/*

# 删除指定日期的知识
rm uploads/auto_knowledge/20260304_*
```

注意：删除文件后，需要重新索引或重启向量数据库。

### Q3: 联网搜索速度慢怎么办？

**原因**：调用外部API需要时间

**优化方法**：
1. 提高相似度阈值，减少联网搜索频率
2. 预先上传常见问题的知识文档
3. 检查网络连接质量

### Q4: 如何验证联网搜索功能是否正常？

**测试步骤**：

1. 查询一个本地知识库绝对没有的问题：
```bash
curl -N -X POST http://localhost:8080/api/query \
  -H "Content-Type: application/json" \
  -d '{"query":"今天的天气如何？","mode":"rag"}'
```

2. 观察返回结果中是否有提示：
```
ℹ️ 本地知识库暂无相关内容，已为您联网搜索最新信息...
```

3. 检查是否生成了新文档：
```bash
ls -lt uploads/auto_knowledge/ | head -1
```
